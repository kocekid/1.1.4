# 一、登录模块

## 1.1 页面定位 / 功能说明

登录模块是用户进入 App 账号体系的入口，负责承接第三方登录、手机号密码登录、邮箱密码登录，以及不同登录方式之间的切换。该模块以登录首页作为聚合入口，用户可直接使用第三方账号登录，也可进入手机号或邮箱登录页完成账号密码登录。

该页面的核心目标：

1. 在登录首页展示品牌、登录引导、协议说明和主登录方式。

3. 支持手机号 + 密码登录，不做短信验证码登录。
4. 支持邮箱 + 密码登录，邮箱可由 Google 登录自动获取，也支持手动设置和更改。
5. 支持不同登录方式之间快速切换。
6. 在表单为空、格式错误、密码缺失、登录失败等场景下给出明确反馈。
7. 登录前需要展示用户协议和隐私政策说明。
8. 登录时需要拦截永久封禁、临时封禁账号，并展示对应封禁提示。
9. 禁止中国大陆 IP 用户登录，命中后拦截登录流程并展示不可登录提示。

## 1.3 前置条件

| 场景 | 前置条件 | 处理方式 |
|---|---|---|
| 首次打开登录页 | 用户未登录或登录状态失效 | 展示登录首页 |

| 手机号登录 | 用户点击手机号登录入口 | 进入手机号密码登录页 |
| 邮箱登录 | 用户点击邮箱登录入口 | 进入邮箱密码登录页 |
| 手机号已输入 | 手机号输入框存在内容 | 展示清空按钮，允许继续输入密码 |
| 邮箱和密码为空 | 用户进入邮箱登录页但未输入字段 | 登录按钮保持禁用或弱态 |
| 登录失败 | 账号不存在、密码错误、第三方授权失败或网络异常 | 展示失败提示，保留必要输入内容 |
| 永久封禁账号登录 | 用户账号状态为永久封禁 | 拦截登录，不进入 App，展示永久封禁提示 |
| 临时封禁账号登录 | 用户账号状态为临时封禁且仍在封禁期内 | 拦截登录，不进入 App，展示临时封禁提示和封禁到期时间 |
| 中国大陆 IP 登录 | 当前登录请求 IP 识别为中国大陆 | 拦截登录，不进入 App，展示当前地区暂不可登录提示 |
| 协议查看 | 用户点击 User Agreement 或 Privacy Policy | 打开对应协议页面 |

## 1.4 页面结构

登录模块由以下页面 / 状态组成：

1. 登录首页
2. 手机号登录页
3. 邮箱登录页
4. 第三方登录入口区
5. 底部登录方式切换区
6. 协议与隐私政策说明区
7. 登录错误 / 授权失败提示状态
8. 账号封禁拦截提示状态
9. 中国大陆 IP 登录拦截状态

| 临时封禁提示 | 用户账号被临时封禁且仍在封禁期内 | 明确提示账号已被临时封禁，并展示封禁到期时间 | 用户只能关闭提示或联系客服，封禁到期前不能进入 App |

## 1.6 统计 / 计算口径

### 1.6.1 登录方式口径

| 登录方式 | 账号来源 | 认证要素 | 处理口径 |
|---|---|---|---|
| 手机号登录 | 用户手动输入手机号 | 手机号 + 密码 | 不做短信验证，只做密码登录 |
| 邮箱登录 | 用户手动输入邮箱，或 Google 登录自动获取邮箱 | 邮箱 + 密码 | 支持手动设置和更改邮箱 |
| Google 登录 | Google 授权返回账号信息 | 第三方授权 | 自动获取邮箱信息 |

| Apple 登录 | Apple 授权返回账号信息 | 第三方授权 | iOS 或支持端展示 |
| Snapchat / TikTok | 第三方授权返回账号信息 | 第三方授权 | 是否开放以产品最终范围为准 |

### 1.6.2 表单校验口径

| 表单 | 校验项 | 处理口径 |
|---|---|---|
| 手机号 | 区号 + 手机号不为空 | 为空时不允许提交 |
| 手机号格式 | 手机号长度和格式符合对应国家规则 | 格式错误时提示手机号格式错误 |
| 手机号密码 | 密码不为空 | 密码为空时不允许提交 |
| 邮箱 | 邮箱不为空且格式正确 | 为空或格式错误时登录按钮不可用或提示格式错误 |
| 邮箱密码 | 密码不为空 | 密码为空时不允许提交 |

### 1.6.3 登录状态口径

| 状态 | 处理口径 |
|---|---|
| 未登录 | 展示登录首页 |
| 登录中 | 按当前登录方式展示加载状态，防止重复提交 |
| 登录成功 | 写入登录状态并进入 App 首页或原目标页面 |
| 登录失败 | 展示失败原因，允许用户重新输入或切换登录方式 |
| 授权取消 | 停留当前登录页，提示用户已取消授权或不提示，按产品方案执行 |
| 登录状态失效 | 引导用户重新登录，成功后回到原目标页面 |
| 永久封禁 | 账号被永久封禁时，登录请求直接拦截，并展示永久封禁提示 |
| 临时封禁 | 账号被临时封禁且未到期时，登录请求直接拦截，并展示封禁到期时间 |
| 中国大陆 IP 拦截 | 当前登录 IP 识别为中国大陆时，登录请求直接拦截 |

## 1.7 交互逻辑

### 1.7.1 登录首页流程

1. 用户打开 App 且未登录。
2. 系统展示登录首页。
3. 页面展示 Logo、品牌名、礼包提示、第三方登录按钮、底部登录方式入口和协议说明。
4. 用户可选择第三方登录，也可点击手机号或邮箱入口进入账号密码登录页。
5. 点击协议或隐私政策时，打开对应协议页面。

### 1.7.2 第三方登录流程

2. 系统拉起对应第三方授权。
3. 用户完成授权。
4. 系统校验第三方账号是否已绑定平台账号。
5. 已绑定时直接登录。
6. 未绑定时按注册 / 资料补全流程处理。
7. 授权失败、取消或网络异常时，返回登录页并展示对应提示。

### 1.7.3 手机号密码登录流程

1. 点击手机号登录入口。
2. 页面进入手机号登录页。
3. 用户选择或确认国家区号。
4. 用户输入手机号。
5. 用户输入密码。
6. 表单满足基本校验后，登录按钮可用。
7. 点击登录按钮。
8. 系统提交手机号和密码进行登录。
9. 登录成功后进入 App 首页或原目标页面。
10. 登录失败时展示失败原因，保留手机号，密码是否保留按安全策略执行。

### 1.7.4 邮箱密码登录流程

1. 点击邮箱登录入口。
2. 页面进入邮箱登录页。
3. 用户输入邮箱地址。
4. 用户输入密码。
5. 邮箱格式正确且密码不为空时，LOGIN 按钮可用。
6. 点击 LOGIN。
7. 系统提交邮箱和密码进行登录。
8. 登录成功后进入 App 首页或原目标页面。
9. 登录失败时展示失败原因，保留邮箱，密码是否保留按安全策略执行。

### 1.7.5 登录方式切换

1. 用户在登录首页点击不同登录方式入口。
2. 页面切换到对应登录页或拉起对应第三方授权。
3. 用户在手机号登录页底部点击邮箱图标时，切换到邮箱登录页。
4. 点击返回按钮时，返回登录首页。
5. 切换登录方式时，已输入内容是否保留按产品最终策略执行。

### 1.7.6 登录限制拦截流程

1. 用户提交手机号、邮箱或第三方授权登录。
2. 系统先识别本次登录请求 IP 所属地区。
3. 若 IP 属于中国大陆，直接拦截登录，展示当前地区暂不支持登录提示，不继续校验账号密码。
4. 若 IP 不属于中国大陆，系统继续校验账号信息。
5. 若账号为永久封禁状态，直接拦截登录，展示永久封禁提示，不进入 App。
6. 若账号为临时封禁状态且当前时间早于封禁到期时间，直接拦截登录，展示临时封禁提示和封禁到期时间，不进入 App。
7. 若账号临时封禁已到期，按账号恢复策略判断是否允许继续登录。
8. 所有拦截场景均不得写入正常登录态，也不得进入首页或原目标页面。

## 1.8 状态定义

| 状态 | 触发条件 | 页面表现 | 用户可操作 |
|---|---|---|---|
| 登录首页默认状态 | 未登录用户进入登录流程 | 展示品牌、第三方登录、底部方式入口和协议说明 | 可选择登录方式 |
| 手机号输入状态 | 用户进入手机号登录页 | 展示区号、手机号、密码输入框 | 可输入手机号和密码 |
| 手机号已输入状态 | 手机号框有内容 | 展示清空按钮 | 可清空或继续输入密码 |
| 邮箱空表单状态 | 用户进入邮箱登录页但未输入字段 | LOGIN 按钮弱态或不可用 | 可输入邮箱和密码 |
| 邮箱格式错误状态 | 邮箱格式不合法 | 展示格式错误提示或按钮不可用 | 可修改邮箱 |
| 登录按钮可用状态 | 必填字段满足基本校验 | 主按钮高亮可点击 | 可提交登录 |
| 登录中状态 | 用户提交登录或第三方授权中 | 展示加载态，按钮防重复点击 | 等待结果 |
| 登录成功状态 | 账号验证通过 | 进入首页或原目标页面 | 正常使用 App |
| 登录失败状态 | 密码错误、账号不存在、授权失败或网络异常 | 展示失败提示 | 可重试或切换登录方式 |
| 永久封禁拦截状态 | 登录账号为永久封禁 | 展示永久封禁提示，不进入 App | 可关闭提示或联系客服 |
| 临时封禁拦截状态 | 登录账号为临时封禁且未到期 | 展示临时封禁提示和封禁到期时间，不进入 App | 可关闭提示或联系客服 |
| 中国大陆 IP 拦截状态 | 登录请求 IP 为中国大陆 | 展示当前地区暂不支持登录提示，不进入 App | 可关闭提示 |
| 协议查看状态 | 用户点击协议或隐私政策 | 打开协议页面 | 可返回登录页 |

## 1.9 系统后台核心逻辑

1. 系统支持手机号密码登录、邮箱密码登录和第三方授权登录。
2. 手机号登录只做密码登录，不做短信验证码登录。
3. 邮箱登录支持通过 Google 登录自动获取邮箱，也支持用户手动设置和更改邮箱。
4. 第三方登录需要校验第三方授权结果，并判断是否已绑定平台账号。
5. 第三方账号未绑定平台账号时，需要进入注册或资料补全流程。
6. 登录成功后，系统需要写入用户登录状态，并返回当前用户基础信息。
7. 如果用户是从受限页面被引导登录，登录成功后应回到原目标页面；没有原目标页面时进入首页。
8. 系统需要对登录失败原因进行明确区分，包括账号不存在、密码错误、账号封禁、授权失败、网络异常等。
9. 登录提交时需要防止重复点击导致重复请求。
10. 用户协议和隐私政策需要按当前语言和地区展示对应版本。
11. 账号存在封禁或不可登录状态时，不允许进入 App，并展示封禁或不可登录原因。
12. 账号被永久封禁时，系统返回永久封禁状态，前端展示永久封禁提示。
13. 账号被临时封禁时，系统返回临时封禁状态和封禁到期时间，前端展示临时封禁提示及到期时间。
14. 登录请求 IP 识别为中国大陆时，系统需要直接拦截登录；手机号、邮箱、第三方登录均适用该限制。

## 1.10 异常与测试关注点

| 场景 | 测试关注点 | 预期结果 |
|---|---|---|
| 登录首页加载 | Logo、第三方按钮、协议是否展示 | 页面展示完整，无空白入口 |
| Google 登录 | 是否能拉起授权并完成登录 | 授权成功后进入 App |

| Apple 登录 | 非支持端是否误展示 | 支持端展示，非支持端隐藏 |

# 二、注册模块

## 2.1 页面定位 / 功能说明

注册模块是新用户完成账号创建后的资料补全页面，负责引导用户设置头像、昵称、性别、生日、国家等基础资料，并支持默认头像、相册上传、拍照等头像来源选择。该模块不是第三方注册选择页，而是注册成功或第三方首次授权后的“完善个人资料”流程。

该页面的核心目标：

1. 让新用户在进入 App 主流程前补齐基础个人资料。
2. 支持用户上传头像、选择默认头像或拍照设置头像。
3. 支持用户输入昵称，并提供清空和随机生成能力。
4. 支持性别、生日、国家等基础资料选择。
5. 国家字段需要承接默认国家策略，且国家列表由服务端返回，客户端不得写死。
6. 用户确认后提交资料，进入 App 首页或下一步注册完成流程。
7. 字段缺失、格式异常、头像权限拒绝等场景需要有明确提示。
8. 禁止中国大陆 IP 用户注册，命中后拦截注册流程并展示不可注册提示。

#### 11.5.6.1 国家选择规则

1. 国家选项由服务端返回，客户端不得写死。

5. 用户选择国家后，返回注册资料补全页并回填国家字段。

#### 11.5.6.2 交互逻辑

1. 点击国家字段，进入选择国家页面。

#### 11.5.6.3 系统后台核心逻辑

1. 系统返回注册资料补全可选国家列表。
2. 国家列表覆盖中东地区、欧美地区国家和印度。
3. 国家列表不得在客户端写死，需支持服务端动态增删与排序调整。

## 2.7 交互逻辑

### 2.7.1 进入资料补全页

1. 用户完成注册或第三方首次授权。
2. 系统先判断注册请求 IP 是否属于中国大陆。
3. 若 IP 属于中国大陆，拦截注册流程，展示当前地区暂不支持注册提示。
4. 若 IP 不属于中国大陆，系统判断该用户是否缺少必要基础资料。
5. 若资料不完整，进入 Complete your profile 页面。
6. 页面默认展示男性头像、随机生成且已去重的昵称、未选择性别、生日和国家字段。
7. 用户逐项完善后点击 Confirm 提交。

### 2.7.2 设置头像流程

1. 点击头像或头像编辑图标。
2. 页面弹出头像选择底部弹层。
3. 用户可选择默认头像、从相册选择或拍照。
4. 若选择相册，系统请求相册权限。
5. 若选择拍照，系统请求相机权限。
6. 用户选择或拍摄头像后，头像预览更新。
7. 点击弹层 Confirm，确认本次头像选择。
8. 若用户关闭弹层未确认，头像保持原状态。

### 2.7.3 编辑昵称流程

1. 点击昵称输入框。
2. 用户手动输入昵称。
3. 点击清空按钮时，清空当前昵称。
4. 点击随机生成按钮时，系统生成一个随机昵称并回填输入框。
5. 用户提交资料时，系统校验昵称合法性。
6. 校验不通过时，展示原因并要求用户修改。

### 2.7.4 选择性别流程

1. 页面展示男性、女性两个选项。
2. 性别默认不选。
3. 点击某个性别选项。
4. 页面切换选中态。
5. 提交资料时保存当前选中性别。
6. 若用户未选择性别就点击 Confirm，Toast 提示“请完善所有注册信息后提交。”。
7. 注册后是否允许修改性别，按个人中心设置模块规则执行。

### 2.7.5 选择生日流程

1. 点击生日字段。
2. 系统打开日期选择器。
3. 用户选择生日并确认。
4. 页面回填所选日期。
5. 提交时系统根据生日计算年龄并执行年龄策略校验。
6. 如果年龄不符合要求，提示用户重新选择或终止注册流程。

### 2.7.6 国家自动选择流程

1. 用户进入注册资料补全页。
2. 系统根据定位 / IP 识别用户所属国家。
3. 若识别结果属于海湾国家，页面自动回填对应国家。
4. 若识别结果不属于海湾国家，系统随机选择 1 个海湾国家作为用户国籍并回填。
5. 若定位 / IP 识别失败，系统随机选择 1 个海湾国家作为默认国籍。
6. 用户提交资料时保存当前国家信息。
7. 注册完成后，用户可在个人中心-个人资料中修改国家，但仅允许成功修改 1 次。

### 2.7.7 提交资料流程

1. 用户完成头像、昵称、性别、生日、国家等资料设置。
2. 点击 Confirm。
3. 系统校验是否所有注册信息均已完成。
4. 若存在未完成项，Toast 提示：“请完善所有注册信息后提交。”。
5. 系统校验昵称合法性和唯一性。
6. 若昵称重复，Toast 提示：“昵称已存在，请重新设置”。
7. 系统校验生日年龄策略、国家合法性和注册 IP 地区限制。
8. 校验通过后提交资料。
9. 保存成功后进入 App 首页或下一步注册完成流程。
10. 保存失败时展示失败原因，保留用户已填写内容。

## 2.9 系统后台核心逻辑

1. 系统判断新用户是否已完成基础资料补全；未完成时进入资料补全流程。
2. 第三方首次授权登录后，如果缺少头像、昵称、性别、生日、国家等必要资料，也需要进入资料补全页。
3. 系统提供默认男性头像、随机生成且已去重的默认昵称、默认生日和默认国家等初始化值；性别默认不选。
4. 国家默认值需要根据定位 / IP 识别生成；识别结果属于海湾国家时自动选择对应国家，识别结果不属于海湾国家或识别失败时随机选择 1 个海湾国家作为用户国籍。
5. 昵称提交时需要校验长度、内容合法性、敏感词和唯一性；若昵称重复，需要返回“昵称已存在，请重新设置”的提示。
6. 默认头像库需要包含 3 个男性头像和 3 个女性头像。
7. 用户上传头像或拍照头像后，系统需要保存头像素材，并按资料审核规则判断是否需要审核。
8. 性别、生日、国家提交后需要写入用户基础资料；若注册后性别禁止调整，需要在后续编辑资料页限制修改。
9. 生日需要用于年龄计算，年龄展示和年龄限制策略都应基于生日计算结果。
10. 提交资料成功后，系统需要标记该用户已完成注册资料补全，避免下次登录重复进入该页面。
11. 提交失败时，不能丢失用户已填写内容；页面应保留当前输入状态，允许用户重试。
12. 注册完成后，系统记录用户国家来源和国家修改次数；后续用户仅允许在个人中心-个人资料中成功修改 1 次国家。
13. 注册请求 IP 识别为中国大陆时，系统需要直接拦截注册；手机号、邮箱、第三方首次授权注册均适用该限制。
14. 用户协议、隐私政策和第三方注册来源不在本页面重复展示，但注册来源需要与账号体系保留绑定关系。

# 三、个人中心设置模块

## 3.1 模块说明

个人中心设置模块用于统一承接账号、密码、隐私、注销、协议和版本更新相关能力。

核心目标：

1. 提供账号设置、修改密码、隐私设置、注销账号、隐私政策、用户协议、检查更新等入口。
2. 支持手机号、邮箱绑定与更改，支持 Google 登录自动获取邮箱。
3. 提供密码修改能力，包括原密码校验、新密码确认、提交结果反馈。
4. 提供隐私设置能力，包括榜单匿名、黑名单管理。
5. 提供账号注销能力，包括资产校验、二次确认、注销成功状态。

## 3.4 修改密码页

### 3.4.1 对应原型

![修改密码](../1.0汇总原型图/个人中心设置-修改密码.jpg)

### 3.4.2 前置条件

| 场景 | 前置条件 | 处理方式 |
|---|---|---|
| 正常进入修改密码页 | 用户已登录且设置了密码 | 展示修改密码页 |
| 首次设置密码 | 用户已登录但未设置过密码 | 展示设置密码页，隐藏原始密码输入框 |
| 未设置密码 | 用户未设置密码（如纯三方登录用户） | 引导用户设置密码 |

### 3.4.3 页面字段定义

| 字段 / 元素 | 展示规则 | 交互规则 | 异常 / 空值处理 |
|---|---|---|---|
| 页面标题 | 展示 `Change Password` | 仅展示 | 无 |
| 返回按钮 | 展示在页面左上方 | 点击返回设置页 | 无 |
| 原始密码输入框 | 用户已设置过密码时展示原始密码输入提示；未设置过密码时隐藏 | 用户输入原始密码 | 密码不可见，可切换显隐；未设置过密码时不展示该输入框 |
| 新密码输入框 | 展示新密码输入提示 | 用户输入新密码 | 密码不可见，可切换显隐 |
| 确认新密码输入框 | 展示确认新密码输入提示 | 用户再次输入新密码 | 密码不可见，可切换显隐 |
| 确认修改按钮 | 展示确认修改按钮 | 点击提交修改密码请求 | 输入不完整时按钮置灰 |

### 3.4.4 交互逻辑

1. 用户从设置页点击修改密码入口进入修改密码页。
2. 系统先判断当前账号是否已设置过密码。
3. 若当前账号已设置过密码，页面展示原始密码、新密码、确认新密码三个输入框和确认修改按钮。
4. 若当前账号未设置过密码，页面隐藏原始密码输入框，仅展示新密码、确认新密码两个输入框和确认修改按钮。
5. 已设置过密码的用户提交后，系统校验原始密码是否正确，并校验新密码与确认新密码是否一致。
6. 未设置过密码的用户提交后，系统不校验原始密码，只校验两次输入密码是否一致。
7. 两次密码一致时，将输入密码设置为该账号密码并保存成功。
8. 保存成功后，提示密码设置成功或修改成功，后续该账号进入已设置密码状态。

### 3.4.5 状态定义

| 状态 | 触发条件 | 页面表现 | 用户可操作 |
|---|---|---|---|
| 修改密码默认状态 | 用户进入修改密码页且已设置过密码 | 展示原始密码、新密码、二次确认新密码输入框和确认修改按钮 | 可输入密码并提交 |
| 首次设置密码默认状态 | 用户进入修改密码页且未设置过密码 | 隐藏原始密码输入框，仅展示新密码、二次确认新密码输入框和确认按钮 | 可输入密码并提交 |
| 原密码错误状态 | 用户提交时原密码校验失败 | 保留输入内容并 Toast 提示"Original password error" | 可重新输入后再次提交 |
| 新密码二次确认不一致状态 | 用户提交时两次新密码不一致 | 保留输入内容并 Toast 提示"New password and confirmation do not match" | 可修改后重新提交 |
| 首次设置密码成功状态 | 未设置过密码的用户提交后校验通过 | Toast 提示“Password set successfully”，并清空输入内容 | 可返回上一页 |
| 密码修改成功状态 | 用户提交后校验通过 | Toast 提示"Password changed successfully"，并清空所有密码输入内容 | 可返回上一页 |

### 3.4.6 系统后台核心逻辑

1. 系统返回当前账号是否已设置过密码的状态标识。
### 3.4.7 异常与测试关注点

| 场景 | 测试关注点 | 预期结果 |
|---|---|---|
| 已设置密码用户进入页面 | 是否展示原始密码输入框 | 展示原始密码、新密码、确认新密码三个输入框 |
| 未设置密码用户进入页面 | 是否隐藏原始密码输入框 | 隐藏原始密码输入框，仅展示新密码、确认新密码两个输入框 |
| 首次设置密码两次不一致 | 是否允许保存成功 | 不允许保存，提示两次密码不一致 |
| 首次设置密码成功 | 是否将输入密码设置为账号密码 | 设置成功，后续进入已设置密码状态 |
| 已设置密码用户原始密码错误 | 是否阻止修改 | 阻止修改并提示原始密码错误 |

2. 用户已设置过密码时，系统校验原始密码是否正确。
3. 用户未设置过密码时，系统不校验原始密码。
4. 系统校验新密码与二次确认新密码是否一致。
5. 密码保存成功后立即生效。
6. 首次设置密码成功后，系统将该账号更新为已设置密码状态。

### 3.5.2 A.2 前置条件

| 国家修改限制提示 | 当账号已用完国家修改次数时展示提示弹窗 | 点击“确定”关闭弹窗 | 无 |

### 3.5.4 A.4 交互逻辑

1. 用户从个人中心设置进入编辑个人资料页。
2. 页面回填当前头像、昵称、性别、生日、国家、个性签名等资料。
3. 用户可修改头像、昵称、生日、国家、个性签名。
4. 性别字段仅展示，不允许修改。
5. 语音签名不在编辑个人资料页展示，也不提供编辑入口。
6. 点击国家字段时，进入选择国家页面。
7. 国家选择范围支持中东地区、欧美地区国家和印度；国家列表由服务端返回，客户端不得写死。
8. 每个账号仅支持按平台规则修改国家；若已达到修改限制，点击国家字段时弹出提示“每个账号仅支持修改1次国家”。
9. 用户点击 Save 按钮后，系统校验资料内容并保存。
10. 保存成功后，同步刷新个人资料页、直播间半屏资料页等用户资料展示场景。

### 3.5.5 A.5 状态定义

| 状态 | 触发条件 | 页面表现 | 用户可操作 |
|---|---|---|---|
| 编辑资料默认状态 | 用户进入编辑资料页 | 展示当前资料和保存按钮 | 可编辑允许修改的字段 |
| 国家未设置状态 | 用户无国家信息 | 国家字段展示占位 | 可点击进入选择国家 |
| 国家修改受限状态 | 用户已达到国家修改次数限制 | 弹出限制提示 | 点击确认关闭提示 |
| 保存成功状态 | 用户提交资料并校验通过 | Toast 提示保存成功并刷新资料 | 可返回上一页 |
| 保存失败状态 | 资料更新失败 | 保留当前输入内容并提示失败 | 可修改后重新提交 |

### 3.5.6 A.6 系统后台核心逻辑

1. 系统返回编辑个人资料页所需的当前资料数据。
2. 系统支持保存头像、昵称、生日、国家、个性签名等资料字段。
3. 性别字段在编辑个人资料页中不可修改。
4. 语音签名字段不在编辑资料页展示。
5. 国家选项由服务端返回，客户端不得写死。
6. 国家选择范围覆盖中东地区、欧美地区国家和印度。
7. 系统记录国家修改次数并在达到限制时拦截再次修改。
8. 资料更新成功后，系统同步刷新个人资料页、直播间半屏资料页等用户资料展示场景。

## 3.7 注销账号页

### 3.7.1 对应原型

![注销账号](../1.0汇总原型图/个人中心设置-注销账号.jpg)

### 3.7.2 前置条件

| 场景 | 前置条件 | 处理方式 |
|---|---|---|
| 正常进入注销账号流程 | 用户已登录 | 展示注销账号入口和说明 |
| 点击 Delete Account | 用户满足进入设置页条件 | 弹出注销确认弹窗 |
| 存在资产或未满足注销条件 | 用户存在资产、提现中订单、未结清预支等 | 拦截注销并提示原因 |

### 3.7.3 页面字段定义

| 字段 / 元素 | 展示规则 | 交互规则 | 异常 / 空值处理 |
|---|---|---|---|
| Delete Account 入口 | 展示注销账号入口 | 点击弹出注销确认弹窗 | 无 |
| 注销说明文案 | 展示注销条件与影响范围 | 仅展示 | 无 |
| 倒计时按钮 | 展示倒计时确认按钮 | 倒计时结束后可点击确认 | 倒计时期间按钮置灰 |
| Cancel 按钮 | 展示取消按钮 | 点击关闭弹窗 | 无 |
| Confirm 按钮 | 倒计时结束后可点击 | 点击提交注销请求 | 条件不满足时拦截 |

### 3.7.4 交互逻辑

1. 用户在设置页点击注销账号，页面弹出二次确认弹窗。
2. 弹窗提示剩余资产、未提现余额、公会 / 工资等风险，并告知注销后不可恢复。
3. 确认按钮需等待倒计时结束后才可点击；点击取消则关闭弹窗。
4. 用户点击确认后，系统执行最终校验；存在资产、提现订单、未结清预支或其他不满足条件时拦截注销。
5. 若存在未结清预支工资，Toast 提示“存在未结清的预支，无法注销账号”。
6. 校验通过后正式注销账号，释放谷歌、苹果、手机号、邮箱等绑定关系，并退出登录。
7. 若用户为公会长，则所属公会自动解散，公会内主播自动退出公会，无需审核。
8. 若用户为主播，则自动退出所在公会，无需审核。
9. 注销联动完成后，系统发送对应系统消息。

### 3.7.5 状态定义

| 状态 | 触发条件 | 页面表现 | 用户可操作 |
|---|---|---|---|
| 注销弹窗状态 | 用户点击注销账号 | 展示二次确认弹窗和倒计时按钮 | 可取消或等 |
| 注销按钮倒计时状态 | 确认按钮等待倒计时结束 | 按钮置灰不可点 | 只能等待或取消 |
| 注销校验失败状态 | 存在资产、订单或其他不满足条件 | 展示不可注销提示 | 处理后重试 |
| 存在未结清预支状态 | 用户提交注销时存在未结清预支工资 | Toast 提示“存在未结清的预支，无法注销账号”，不执行注销 | 结清预支后重试 |
| 注销成功状态 | 用户完成注销 | 提示账号已注销并退出登录 | 不可继续使用原账号 |

### 3.7.6 系统后台核心逻辑

1. 系统返回注销账号页所需的说明文案和注销条件。
2. 系统校验用户金币、钻石、可提现薪资是否为 0。
3. 系统校验用户是否存在申请提现中订单。
4. 系统校验用户是否存在未结清预支工资。
5. 用户为公会长时，系统自动解散所属公会，并使公会内主播自动退出公会，无需审核。
6. 用户为主播时，系统自动退出所在公会，无需审核。
7. 注销成功后，系统清除用户登录状态并跳转至登录页。
8. 系统同步发送因注销引发的公会解散 / 主播退出系统消息。

### 3.7.7 异常与测试关注点

| 场景 | 测试关注点 | 预期结果 |
|---|---|---|
| 账户存在资产 | 用户仍有金币、钻石、可提现薪资时是否允许注销 | 不允许注销，提示先处理资产 |
| 存在申请提现中订单 | 用户仍有申请提现中订单时是否允许注销 | 不允许注销，提示先完成提现流程 |
| 未结清预支工资 | 用户存在未结清预支时是否允许注销 | 不允许注销，Toast 提示“存在未结清的预支，无法注销账号” |
| 公会长注销 | 用户为公会长时是否自动解散公会并同步主播退出 | 公会自动解散，主播自动退出，无需审核，并发送对应系统消息 |
| 主播注销 | 用户为主播时是否自动退出所在公会 | 自动退出所在公会，无需审核，并通知公会长 |
| 校验全部通过 | 用户满足全部注销条件时是否成功注销 | 注销成功，清除登录状态并退出登录 |

## 3.8 跨页面通用规则

### 3.8.1 协议页面跳转规则

1. 点击隐私政策或用户协议入口时，打开对应协议页面。
2. 协议页面支持多语言，按用户当前语言设置展示。

### 3.8.2 版本检查规则

1. 点击检查更新时，系统校验当前版本是否有新版本。
2. 有新版本时，引导用户前往应用商店更新。
3. 无新版本时，提示“已是最新版本”。

# 四、首页模块

## 4.1 页面定位 / 功能说明

首页是用户进入 App 后的主入口页面，负责承接热门房间曝光、运营活动曝光、财富用户展示、房间列表浏览、国家 / 地区筛选、搜索、消息入口、新人礼包弹窗等核心能力。

首页的核心目标：

1. 让用户快速看到当前热门房间，并进入房间。
2. 通过运营 Banner 承接活动、礼包、充值、权益等运营入口。
3. 展示本周高价值用户 / 财富榜用户，提升社交氛围和付费用户曝光。
4. 通过国家 / 地区筛选，让用户快速定位感兴趣的房间。
5. 对新注册用户发放新人礼包，并通过弹窗完成奖励确认。

### 4.6.1 Hot Rooms 展示口径

| 规则项 | 口径 |
|---|---|
| 数据来源 | 本周房间榜单 |

| 交互方式 | 支持左右滑动查看不同名次 |
| 黑名单处理 | 支持配置房间黑名单，被配置房间不展示 |

### 4.6.2 Big shot 展示口径

| 规则项 | 口径 |
|---|---|
| 数据来源 | 本周财富榜单 |
| 展示范围 | 前 5 个用户 |
| 更新频率 | 每隔 60 分钟更新一次 |
| 点击行为 | 点击用户头像进入用户个人资料页 |

### 4.6.3 房间列表排序口径

| 排序类型 | 规则 |
|---|---|
| 热力值排序 | 房间列表按房间热力值降序展示 |
| 自动排序更新频率 | 每隔 60 分钟更新一次 |

房间热力值计算公式：

```text
房间热力值 = 房间当前在线人数 × 0.4 + 房间 60 分钟内金币流水 × 0.6
```

幸运礼物金币需要先按以下规则换算：

```text
幸运礼物计入金币流水的数值 = 幸运礼物金币数 ÷ 10
```

### 4.6.4 房间展示过滤规则

首页房间相关曝光位采用「存在任一条件即不展示」原则。

| 过滤条件 | 过滤规则 | 说明 |
|---|---|---|
| 当前用户黑名单命中 | 不展示该房间 | 用户维度过滤 |
| 后台封禁房间 | 不展示该房间 | 平台维度过滤 |
| 后台隐藏房间 | 不展示该房间 | 平台维度过滤 |
| 其他不可展示条件 | 不展示该房间 | 包含已失效、已删除或业务配置禁止曝光等情况 |

> **过滤原则**：以上条件为并列关系，命中任一条件即不展示，无需继续参与热力值排序、人工排序、热门房间过滤或其他曝光计算。

> **生效时效**：房间封禁/隐藏实时生效，不依赖首页 60 分钟业务刷新周期；已展示房间若新增封禁或隐藏状态，需立即从首页所有曝光位移除。

### 4.6.5 热门房间过滤房间列表口径

当用户在 Hot Rooms 中选中某个热门房间名次时，房间列表需要过滤该名次及相邻名次房间。

示例：

| 选中名次 | 需要隐藏的名次 | 房间列表展示 |
|---|---|---|
| 第 3 名 | 第 1 名、第 3 名、第 4 名 | 展示第 2 名、第 5 名及其他非隐藏房间 |

说明：脑图示例明确为“滑动选择第 3 名，则隐藏 1、3、4 名次房间，展示 2、5 名次房间”。该规则按原始脑图落地，后续如果产品口径调整，需要统一修正。

## 4.7 交互逻辑

### 4.7.1 首页初始化

1. 用户进入首页。
2. 系统加载顶部用户信息、Banner、热门房间、财富用户、国家 / 地区筛选项、房间列表。
3. 若用户为新注册且符合新人礼包条件（注册设备下无其他账号），优先展示礼品动效。
4. 礼品动效播放完成后，展示 VIP1 3 天奖励弹窗。
5. 点击“收下”后关闭弹窗，回到首页主界面。

### 4.7.2 Banner 交互

1. 点击 Banner。
2. 系统判断 Banner 当前配置的跳转目标是否有效。
3. 若有效，进入对应活动页、礼包页、充值页或其他承接页。
4. 若无效，停留当前页并提示“活动暂不可用”。

### 4.7.3 Hot Rooms 交互

1. 首页展示本周房间榜单前 5 房间。
2. 用户左右滑动 Hot Rooms 卡片，查看不同名次房间。
3. Hot Rooms 默认按 2、1、3、4、5 的视觉顺序展示。
4. 点击任一热门房间卡片。
5. 系统判断房间是否可进入：
   - 可进入：进入房间。
   - 锁房或权限限制：展示对应进入限制提示或校验入口。
   - 房间已关闭：提示“房间暂不可进入”，并刷新热门房间数据。
6. 用户选中某个热门房间名次后，房间列表按过滤规则隐藏该名次及相邻名次房间。

### 4.7.4 Big shot 交互

1. 首页展示本周财富榜前 5 用户。
2. 点击某个用户头像或昵称。
3. 跳转到该用户个人资料页。
4. 若用户状态异常或资料不可访问，展示对应不可访问提示。

### 4.7.5 国家 / 地区筛选交互

1. 默认选中 All，展示全部房间列表。
2. 点击某个国家 / 地区标签。
3. 房间列表刷新为该国家 / 地区下的房间。
4. 用户横向滑动标签栏，可查看更多国家 / 地区。
5. 若筛选后无房间，展示空状态。

### 4.7.6 房间列表交互

1. 用户浏览房间列表。
2. 房间列表按热力值排序结果展示房间。
3. 点击房间卡片、封面或房间名称。
4. 系统判断房间进入条件：
   - 普通房间：直接进入房间。
   - 锁房：触发密码或权限校验。
   - 房间已关闭或被回收：提示房间暂不可进入，并刷新列表。
5. 点击更多按钮，展示更多操作入口。

## 4.9 系统后台核心逻辑

1. 系统按运营配置生成首页 Banner 展示内容，包括图片、文案、跳转目标和有效期。

2. 系统每隔 60 分钟刷新一次 Hot Rooms 数据，取本周房间榜单前 5 个可展示房间。

3. Hot Rooms 需要排除被配置为不展示的房间，避免黑名单房间进入首页曝光。

4. 系统每隔 60 分钟刷新一次 Big shot 数据，取本周财富榜前 5 个用户，并提供用户资料页跳转能力。

5. 系统每隔 60 分钟刷新一次房间列表排序结果。

6. 房间列表按房间热力值降序展示。

7. 房间热力值由当前在线人数和最近 60 分钟金币流水共同计算，幸运礼物金币需要按 10 比 1 折算后再计入流水。

8. 系统需要判定新注册用户是否符合新人礼包领取条件：注册设备下无其他账号且当前账号为首次登录。

9. 新人礼包发放成功后，系统发放 VIP1 3 天权益，并同步发送站内信：恭喜来到 TOP KHALIJ，平台已赠送你 3 天 VIP 会员供体验，祝您玩得愉快！

10. 新人礼包的设备维度限制：同一注册设备下仅首个符合条件的新账号可领取一次；设备下已有其他账号时，新注册账号不具备领取资格。

## 4.10 异常与测试关注点

| 场景 | 测试关注点 | 预期结果 |

# 五、搜索模块

## 5.1 页面定位 / 功能说明

搜索模块是用户在 App 内查找房间和用户的统一入口，负责承接首页搜索入口、顶部搜索框输入、Room / User 分类切换、搜索结果展示和结果点击跳转。

该页面的核心目标：

## 5.7 交互逻辑

### 5.7.1 进入搜索页

1. 用户从首页或其他搜索入口进入搜索页。
2. 页面展示返回按钮、搜索输入框、Search 按钮、Room / User 分类 Tab。
3. 默认选中的 Tab 以入口配置为准；若无明确配置，默认进入 Room 或上次用户使用的分类。
4. 搜索词为空时，结果区不展示搜索结果。

### 5.7.2 输入与清空搜索词

1. 点击搜索输入框。
2. 输入用户 ID、靓号、房间 ID、房间名称或昵称关键词。
3. 输入框内展示清空按钮。
4. 点击清空按钮后，搜索词清空，结果区恢复默认状态。
5. 清空后再次点击 Search 不发起搜索，提示请输入搜索内容或保持按钮不可用。

### 5.7.3 用户搜索流程

1. 用户切换到 User 分类。
2. 输入用户 ID、靓号或用户关键词。
3. 点击 Search。
4. 系统查找匹配用户。
5. 若存在结果，展示用户结果卡片。
6. 点击某个用户结果卡片。
7. 系统进入该用户个人资料页。
8. 若用户资料不可访问，展示不可访问提示。

### 5.7.4 房间搜索流程

1. 用户切换到 Room 分类。
2. 输入房间 ID、房间名称、用户 ID 或靓号。
3. 点击 Search。
4. 系统查找匹配房间。
5. 若存在结果，展示房间结果卡片。
6. 点击某个房间结果卡片。
7. 系统校验房间当前状态。
8. 房间可进入时进入房间；不可进入时展示对应提示。

### 5.7.5 Room / User 切换

1. 用户在搜索页点击 Room 或 User。
2. 页面切换 Tab 选中态。
3. 当前搜索词保持不变。
4. 若当前搜索词已有搜索结果缓存，可直接展示对应分类缓存结果。
5. 若无缓存，则按当前搜索词请求对应分类结果。
6. 切换失败或加载失败时，保留原搜索词和当前页面，并提示加载失败。

## 5.8 状态定义

| 状态 | 触发条件 | 页面表现 | 用户可操作 |
|---|---|---|---|
| 默认空白状态 | 刚进入搜索页且未输入搜索词 | 展示搜索框、分类 Tab，结果区为空或展示引导文案 | 可输入搜索词 |
| 输入中状态 | 用户正在输入搜索词 | 输入框展示内容和清空按钮 | 可继续输入、清空或搜索 |
| User 结果状态 | User 分类下搜索成功且有结果 | 展示用户结果卡片 | 可点击进入用户资料页 |
| Room 结果状态 | Room 分类下搜索成功且有结果 | 展示房间结果卡片 | 可点击进入房间 |
| 无结果状态 | 当前分类无匹配结果 | 展示无结果提示 | 可修改搜索词或切换分类 |
| 加载中状态 | 点击 Search 或切换 Tab 后请求中 | 展示加载态 | 等待加载完成 |
| 网络异常状态 | 搜索请求失败 | 展示失败提示 | 可重试 |
| 房间不可进入状态 | 点击房间结果但房间关闭、封禁、已满、锁房或权限不足 | 展示不可进入原因 | 可返回搜索结果继续浏览 |
| 用户资料不可访问状态 | 点击用户结果但用户异常、注销、封禁或资料不可见 | 展示不可访问提示 | 可返回搜索结果继续浏览 |

## 5.9 系统后台核心逻辑

1. 系统支持用户搜索和房间搜索两类结果，并按 Room / User 分类独立返回。
2. 用户搜索需要支持用户 ID、靓号和昵称关键词匹配。
3. 房间搜索需要支持房间 ID、房间名称、用户 ID 和靓号匹配。
4. 当搜索词精确命中用户 ID 或靓号时，对应用户应优先展示。
5. 当搜索词命中用户 ID 或靓号且该用户有关联房间时，Room 分类需要支持展示该用户关联的房间。
6. 搜索结果返回前，需要过滤用户无权限查看的用户资料和无权限进入的房间；如需要保留展示，则点击时必须再次校验。
7. 房间结果点击进入前，需要实时校验房间状态，避免进入关闭、封禁、回收、已满或权限不足的房间。
8. 搜索结果中的用户 ID 展示需要遵循全局用户 ID 统一规则：用户有靓号时按产品规则展示靓号或同时展示原始 ID。
9. 系统记录搜索行为所需的基础日志，用于问题排查和搜索质量优化；正式埋点口径如有单独埋点方案，以埋点文档为准。

## 5.10 异常与测试关注点

| 场景 | 测试关注点 | 预期结果 |
|---|---|---|
| 空搜索词点击 Search | 是否发起无效搜索 | 不发起搜索，按钮不可用或提示请输入搜索内容 |
| 输入用户靓号搜索用户 | 是否能找到持有该靓号的用户 | User 分类展示目标用户 |
| 输入用户 ID 搜索用户 | 是否能找到对应用户 | User 分类展示目标用户 |
| 输入用户 ID 搜索房间 | 是否能找到该用户关联房间 | Room 分类展示关联房间 |
| 输入房间名称搜索 | 是否能找到匹配房间 | Room 分类展示匹配房间 |
| Room / User 切换 | 搜索词是否保留 | 切换后搜索词不清空，结果按分类刷新 |
| 搜索无结果 | 是否展示清晰空状态 | 当前分类展示无结果提示 |
| 用户头像为空 | 是否展示默认头像 | 用户结果卡片不破图 |
| 房间封面为空 | 是否展示默认房间封面 | 房间结果卡片不破图 |
| 房间不可进入 | 点击不可进入房间 | 不进入房间，展示不可进入原因 |
| 用户资料不可访问 | 点击异常用户结果 | 不进入资料页，展示不可访问提示 |
| 网络异常 | 搜索请求失败 | 展示错误提示，可重新搜索 |

## 6.7 交互逻辑

### 6.7.1 进入我的房间页

1. 用户进入我的房间页面。
2. 页面加载顶部导航、创建房间卡片、房间分类 Tab 和默认列表。
3. 默认选中 Recent。
4. 系统加载用户最近访问房间列表。
5. 若列表为空，展示 Recent 空状态。

### 6.7.2 创建房间交互

1. 点击 Create room 卡片或右侧加号按钮。
2. 系统判断用户是否具备创建房间权限。
3. 若具备权限，系统直接使用当前用户名称作为默认房间名称，并使用当前用户默认头像或系统默认头像作为房间初始头像 / 封面创建房间。
4. 房间创建成功后，用户自动进入新创建的房间。
5. 若不具备权限，展示不可创建原因。
6. 用户后续返回我的房间页面时，展示已创建房间信息。

### 6.7.3 已创建房间展示交互

1. 用户已有房间时，页面展示已创建房间的封面、名称和房间说明。
2. 点击已创建房间卡片。
3. 系统判断房间当前状态。
4. 若房间可进入，进入房间。
5. 若房间被封禁、回收、关闭或不可进入，展示对应提示。

### 6.7.4 房间分类 Tab 切换

1. 点击 Recent、Following 或 Favorite。
2. 页面切换选中态。
3. 系统加载对应分类下的房间列表。
4. 若加载成功，展示对应列表。
5. 若列表为空，展示对应分类的空状态。
6. 若加载失败，保留当前页面并提示加载失败，可重试。

### 6.7.5 关注房间列表交互

1. 点击 Following。
2. 系统展示用户已关注房间列表。
3. 列表按关注时间降序排列。
4. 点击房间卡片进入房间。
5. 若房间已取消关注或已失效，刷新列表并提示房间不可用。

### 6.7.6 收藏房间列表交互

1. 点击 Favorite。
2. 系统展示用户收藏房间列表。
3. 点击房间卡片进入房间。
4. 若收藏房间已失效，刷新列表并提示房间不可用。

## 6.9 系统后台核心逻辑

1. 系统判断当前用户是否已有创建的房间。
2. 用户未创建房间时，页面保留创建房间入口。
3. 点击创建房间后，系统需要直接创建房间，不增加额外填写步骤；默认房间名称取当前用户名称，默认房间头像 / 封面取当前用户默认头像或系统默认头像。
4. 房间创建成功后，系统需要让用户自动进入新房间，并把该房间写入用户已创建房间信息。
5. 用户已创建房间时，系统返回该房间的封面、名称和房间说明，用于页面展示。
6. 系统记录用户最近访问过的房间，并在 Recent 分类中展示。
7. 系统记录用户关注的房间，并在 Following 分类中展示。
8. Following 分类中的房间按关注时间降序排序。
9. 系统记录用户收藏的房间，并在 Favorite 分类中展示。
10. 房间列表展示前需要过滤已失效、已回收、不可见或用户无权限访问的房间。
11. 点击房间时，系统需要再次校验房间是否可进入，避免用户进入已失效房间。
12. 房间封面支持动效展示时，应按全局动效内容规则展示；不支持或加载失败时展示静态图或默认图。

# 七、榜单模块

## 7.1 页面定位 / 功能说明

榜单模块是 App 内集中展示用户、房间、关系类排名的页面，用于承接财富榜、魅力榜、房间榜等排行榜能力，并通过日榜、周榜、月榜切换展示不同周期下的排名结果。

该页面的核心目标：

1. 展示不同榜单类型下的排名结果。
2. 支持日榜、周榜、月榜切换。
3. 突出展示榜单前列用户或房间，强化荣誉感和付费 / 活跃激励。
4. 展示当前用户自己的排名信息，让用户知道自己距离上榜或更高名次的差距。
5. 为榜单展示好友设置、榜单隐身、用户资料跳转等能力提供入口。

| 周期 | 统计范围 | 刷新方式 | 展示说明 |
|---|---|---|---|
| 日榜 | 当日榜单统计周期 | 按榜单刷新策略更新 | 展示当天排名 |
| 周榜 | 本周榜单统计周期 | 按榜单刷新策略更新 | 展示本周排名 |
| 月榜 | 本月榜单统计周期 | 按榜单刷新策略更新 | 展示本月排名 |

### 7.6.3 排序口径

| 规则项 | 口径 |
|---|---|
| 财富榜排序 | 按财富值降序排序 |
| 魅力榜排序 | 按魅力值降序排序 |
| 房间榜排序 | 按房间流水降序排序 |
| 同分排序 | 同分数时，先达到该分数的用户或房间排名靠前 |
| 前列展示范围 | 原型展示第一至第六名，普通列表从后续名次开始展示 |
| 当前用户排名 | 当前用户不在前列展示范围时，仍可在当前用户排名提示区展示自己的排名和榜单值 |

### 7.6.4 榜单隐身口径

| 规则项 | 口径 |
|---|---|
| 展示条件 | 用户当前会员等级或权益允许榜单隐身时，展示显示 / 隐藏榜单入口 |
| 隐身范围 | 隐身只影响榜单对外展示，不改变真实统计和排名计算 |
| 切换结果 | 用户切换后，榜单页应按最新展示状态刷新 |
| 权益不足 | 用户不具备权益时，不展示入口或提示需要对应权益 |

## 7.7 交互逻辑

### 7.7.1 进入榜单页

1. 用户从榜单入口进入页面。
2. 页面加载默认榜单类型和默认周期。
3. 系统返回前列排名、当前用户排名和普通列表数据。
4. 页面优先展示前列排名展示区，再展示普通排名列表。
5. 若当前榜单无数据，展示榜单空状态。

### 7.7.2 切换榜单类型

1. 点击财富榜、魅力榜、房间榜或其他已开放榜单类型。
2. 页面切换榜单类型选中态。
3. 系统按当前选中周期加载对应榜单数据。
4. 加载成功后刷新前列展示区、当前用户排名提示区和普通列表。
5. 加载失败时保留切换前数据，并提示加载失败。

### 7.7.3 切换榜单周期

1. 点击日榜、周榜或月榜。
2. 页面切换周期选中态。
3. 系统按当前榜单类型和新周期重新加载榜单数据。
4. 加载成功后刷新排名结果。
5. 当前用户排名提示区同步更新为新周期下的排名。

### 7.7.4 点击榜单对象

1. 点击前列排名展示位或普通列表项。
2. 系统判断当前榜单对象类型。
3. 如果是用户榜对象，进入对应用户个人资料页。
4. 如果是房间榜对象，进入对应房间或房间详情。
5. 若对象已失效、不可见或当前用户无权限访问，展示不可访问提示。

### 7.7.5 当前用户排名操作

1. 页面展示当前用户在当前榜单和周期下的排名提示。
2. 点击当前排名提示区。
3. 若产品支持定位榜单位置，则列表滚动到当前用户排名附近。
4. 若当前用户未上榜，则展示未上榜说明和提升引导。
5. 若用户具备榜单隐身权益，可通过显示 / 隐藏入口切换榜单展示状态。

### 7.7.6 榜单展示好友入口

1. 用户在财富榜、魅力榜或房间榜进入前三名。
2. 若当前名次支持设置榜单展示好友，前列展示位出现添加或更换入口。
3. 点击后进入展示好友设置流程。
4. 展示好友的搜索、添加、确认、更换、付费规则不在本模块展开，统一在“榜单展示好友设置模块”中定义。

## 7.8 状态定义

| 状态 | 触发条件 | 页面表现 | 用户可操作 |
|---|---|---|---|
| 加载中 | 进入页面、切换榜单类型或周期 | 展示加载态 | 等待加载完成 |
| 正常有数据 | 当前榜单类型和周期有排名数据 | 展示前列排名、当前用户排名、普通列表 | 可切换榜单、周期、点击对象 |
| 榜单空状态 | 当前榜单无排名数据 | 展示空状态提示 | 可切换其他榜单或周期 |
| 当前用户未上榜 | 当前用户无排名或排名不在展示范围内 | 当前用户排名区展示未上榜提示 | 可继续浏览榜单 |
| 用户榜对象不可访问 | 点击用户榜对象时用户封禁、注销或资料不可见 | 展示不可访问提示 | 返回榜单继续浏览 |
| 房间榜对象不可进入 | 点击房间榜对象时房间关闭、封禁、回收、已满或权限不足 | 展示不可进入原因 | 返回榜单继续浏览 |
| 榜单隐身可用 | 用户具备对应权益 | 展示显示 / 隐藏入口 | 可切换榜单展示状态 |
| 榜单隐身不可用 | 用户不具备对应权益 | 不展示入口或提示需要权益 | 可继续浏览榜单 |
| 网络异常 | 榜单数据加载失败 | 展示错误提示 | 可重试 |

## 7.9 系统后台核心逻辑

1. 系统需要按榜单类型和榜单周期生成排名结果。
2. 财富榜按用户财富值降序排序。
3. 魅力榜按用户魅力值降序排序。
4. 房间榜按房间流水降序排序。
5. 榜单展示过滤规则如下：
   - 财富榜、魅力榜：用户被封禁时，从对应榜单移除。
   - 房间榜：房间被封禁或隐藏时，从房间榜移除该房间。
   - 房间榜：房主被封禁时，房间榜同步移除该房间。
   - 房间榜：房间内成员被封禁，不影响房间榜排名结果。
6. 榜单过滤优先级高于榜单排序、榜单名次展示和前列展示逻辑。
封禁/隐藏状态实时生效，不依赖榜单业务刷新周期；已上榜对象若在展示后变为封禁/隐藏状态，需立即从对应榜单移除。
5. 同分数时，先达到该分数的用户或房间排名靠前。
6. 系统需要分别返回前列展示数据、普通列表数据和当前用户排名数据，保证页面能同时展示头部荣誉区和列表区。
7. 当前用户未进入前列展示区时，系统仍需提供当前用户在当前榜单和周期下的排名状态。
8. 榜单隐身只改变对外展示状态，不改变真实榜单统计、排名计算和奖励结算口径。
9. 当前用户具备榜单隐身权益时，系统返回当前展示状态，用于页面显示“显示 / 隐藏”入口。
10. 榜单对象点击前需要校验对象是否仍可访问；用户异常或房间异常时，不允许进入并返回对应提示。
11. 榜单展示好友资格由榜单名次、榜单类型和用户历史设置状态共同决定；具体设置流程在展示好友设置模块中定义。

|---|---|---|

| 网络异常 | 榜单加载失败 | 展示错误提示并支持重试 |

# 八、榜单展示好友设置模块

## 8.1 页面定位 / 功能说明

榜单展示好友设置模块用于让满足榜单高位条件的用户，在自己的榜单展示位中添加或更换一名好友，共享榜单荣誉展示。该模块承接榜单页前列展示位上的添加入口、更换入口、首次引导弹窗、好友搜索 / 关注列表选择、确认展示、免费次数与付费更换等完整流程。

该模块的核心目标：

1. 仅允许符合条件的榜单高位用户设置榜单展示好友。
2. 支持从关注列表中选择好友，也支持通过用户 ID 搜索指定用户。
3. 首次获得可设置资格时，需要给用户明确引导。
4. 设置前需要二次确认，避免误操作。
5. 已设置后支持更换，并按榜单名次和历史次数判断是否免费或需要消耗金币。
6. 设置成功后，榜单展示位需要同步刷新为已展示好友状态。

## 8.6 统计 / 计算口径

### 8.6.1 可设置资格口径

| 规则项 | 口径 |
|---|---|
| 支持榜单 | 财富榜、魅力榜、房间榜 |
| 支持名次 | 仅 TOP3 用户可设置榜单展示好友 |
| 非 TOP3 用户 | 不展示添加或更换入口 |
| 资格变化 | 用户从 TOP3 掉出后，不允许继续新增或更换展示好友；已展示内容是否保留需按产品最终策略确认 |

### 8.6.2 免费次数口径

| 名次 | 免费机会 | 后续设置 / 更换口径 |
|---|---|---|
| TOP1 | 每个用户在对应榜单成为 TOP1 后，终身只有 1 次免费设置榜单展示好友机会 | 免费机会用完后，每次设置或更换需要扣除对应金币 |
| TOP2 / TOP3 | 无固定免费首免，或按配置判断是否存在免费机会 | 后续每次设置或更换需要扣除对应金币 |
| 配置金币为 0 | 视为可直接设置或更换 | 不扣除金币 |

说明：脑图明确写到“每个用户在房间榜单、财富榜单、魅力榜单成为 TOP1 后，终身只有 1 次免费设置榜单展示好友机会”；TOP2、TOP3 后续每次设置需要扣除对应金币。若 TOP2 / TOP3 是否存在首次免费机会需要进一步确认，当前按“配置控制”处理。

### 8.6.3 付费更换口径

| 规则项 | 口径 |
|---|---|
| 金币数来源 | 由服务端按榜单类型、名次或统一配置返回 |
| 扣费时机 | 用户确认支付 / 确认更换后扣除 |
| 扣费成功 | 进入好友选择或完成展示好友替换 |
| 扣费失败 | 不改变原展示好友，提示失败原因 |
| 金币不足 | 提示余额不足，允许用户取消或进入充值入口，具体以产品最终方案为准 |

### 8.6.4 首次提示口径

| 规则项 | 口径 |
|---|---|
| 触发条件 | 用户首次成为财富榜、魅力榜或房间榜 TOP3 |
| 判断方式 | 系统需要记录用户在对应榜单是否已展示过首次提示 |
| 弹窗行为 | 首次触发时展示引导弹窗，用户可点击去设置 |
| 重复展示 | 同一用户、同一榜单的首次提示不应无限重复弹出 |

## 8.7 交互逻辑

### 8.7.1 首次添加展示好友流程

1. 用户进入榜单页。
2. 系统判断用户是否为财富榜、魅力榜或房间榜 TOP3。
3. 若用户首次获得 TOP3 资格，展示首次引导弹窗。
4. 点击“添加朋友”。
5. 页面打开添加好友展示弹窗。
6. 用户可从关注列表选择好友，也可输入用户 ID 搜索好友。
7. 点击候选用户右侧“添加”。
8. 系统展示确认展示好友弹窗。
9. 点击确认。
10. 系统校验用户榜单资格、免费次数、好友状态和展示位状态。
11. 校验通过后保存展示好友关系。
12. 榜单展示位刷新为已展示好友状态。

### 8.7.2 从关注列表添加流程

1. 用户打开添加好友展示弹窗。
2. 弹窗展示关注列表。
3. 用户选择某个关注用户并点击添加。
4. 系统判断该用户是否可作为展示好友。
5. 可添加时进入确认展示好友弹窗。
6. 用户确认后完成设置。
7. 不可添加时，展示不可添加原因。

### 8.7.3 搜索用户添加流程

1. 用户在添加好友展示弹窗输入用户 ID。
2. 点击搜索。
3. 系统按用户 ID 查找目标用户。
4. 找到用户时展示搜索结果。
5. 点击添加。
6. 进入确认展示好友弹窗。
7. 用户确认后完成设置。
8. 未找到用户时，展示无搜索结果提示。

### 8.7.4 已设置后的更换流程

1. 用户在榜单展示位看到已设置的展示好友。
2. 点击更换入口。
3. 系统判断当前用户是否仍具备该榜单展示好友更换资格。
4. 系统判断是否还有免费次数。
5. 若仍可免费更换，进入添加好友展示弹窗。
6. 若需要付费更换，展示消耗金币确认弹窗。
7. 用户确认支付 / 确认更换。
8. 系统扣除金币或确认免费机会消耗。
9. 进入添加好友展示弹窗，用户选择新好友。
10. 用户确认后替换原展示好友。
11. 更换成功后榜单展示位刷新。

### 8.7.5 取消与失败处理

1. 用户在任意确认弹窗点击取消，关闭当前弹窗，不改变原展示好友。
2. 用户在添加好友展示弹窗关闭页面，返回榜单页，不保存未确认选择。
3. 提交设置失败时，展示失败原因，保留原展示状态。
4. 扣费失败时，不进入更换流程，不改变展示好友。
5. 好友账号异常时，不允许设置并提示该用户暂不可展示。

## 8.8 状态定义

| 状态 | 触发条件 | 页面表现 | 用户可操作 |
|---|---|---|---|
| 无资格状态 | 用户未进入对应榜单 TOP3 | 不展示添加 / 更换入口 | 可浏览榜单 |
| 有资格未设置状态 | 用户进入 TOP3 且当前榜位未设置好友 | 榜位展示添加按钮 | 可点击添加展示好友 |
| 首次引导状态 | 用户首次成为对应榜单 TOP3 | 展示首次引导弹窗 | 可点击添加朋友或关闭 |
| 添加弹窗状态 | 用户点击添加入口 | 展示用户 ID 搜索框和关注列表 | 可搜索、选择、添加 |
| 搜索无结果状态 | 输入用户 ID 后未找到用户 | 展示无结果提示 | 可修改 ID 重新搜索 |
| 确认展示状态 | 用户选择候选好友后 | 展示确认展示好友弹窗 | 可取消或确认 |
| 已设置状态 | 用户已设置展示好友 | 榜位展示好友信息和更换入口 | 可查看或更换 |
| 免费更换状态 | 用户仍有免费机会 | 更换时不扣金币 | 可直接进入选择流程 |
| 付费更换状态 | 免费机会已用完且配置金币大于 0 | 展示消耗金币确认弹窗 | 可取消或确认支付 |
| 金币不足状态 | 付费更换时用户金币不足 | 展示余额不足提示 | 可取消或前往充值 |
| 设置失败状态 | 保存展示好友失败 | 展示失败原因 | 可重试或取消 |

## 8.9 系统后台核心逻辑

1. 系统需要按榜单类型、榜单周期和用户当前名次判断是否具备展示好友设置资格。
2. 只有财富榜、魅力榜、房间榜 TOP3 用户可以设置榜单展示好友。
3. 系统记录用户在每个榜单上的首次提示状态，避免首次引导弹窗反复出现。
4. 系统记录每个用户在对应榜单、对应名次下的免费设置次数使用情况。
5. TOP1 的免费机会需要按“终身 1 次”控制，不能因周期变化重复获得。
6. TOP2 / TOP3 的设置和更换费用按服务端配置执行，配置为 0 时可直接设置。
7. 用户提交设置前，系统需要再次校验榜单资格、展示位状态、候选好友状态和金币余额。
8. 付费更换时，扣费成功后才允许进入或完成更换流程；扣费失败不得改变原展示好友。
9. 设置成功后，系统需要保存展示好友关系，并在榜单页返回最新展示状态。
10. 如果被展示好友账号注销、封禁或不可见，系统支持隐藏、置灰或清空展示状态，具体展示策略需产品确认。
11. 榜单展示好友只影响展示关系，不改变榜单排名、榜单值、奖励统计和结算口径。
9. 若被设置为随同上榜展示的好友本人已开启榜单匿名，系统在榜单好友展示位仍返回该好友真实头像，不返回匿名头像；该例外仅作用于好友展示位，不改变该好友在榜单主体中的匿名状态。

## 8.10 异常与测试关注点
| 被设置好友开启榜单匿名 | 榜单好友展示位是否仍显示真实头像 | 展示该好友真实头像，不使用匿名头像 |

| 场景 | 测试关注点 | 预期结果 |
|---|---|---|
| 非 TOP3 用户进入榜单页 | 是否误展示添加入口 | 不展示添加或更换入口 |
| TOP3 未设置好友 | 是否展示添加按钮 | 对应榜位展示添加入口 |
| 首次成为 TOP3 | 是否展示首次引导弹窗 | 展示引导弹窗且可点击去设置 |
| 首次引导重复触发 | 同一榜单是否反复弹窗 | 已提示过后不重复弹出 |
| 从关注列表添加好友 | 是否能进入确认弹窗 | 点击添加后展示确认展示好友 |
| 通过用户 ID 搜索好友 | 是否能找到正确用户 | 展示目标用户并可添加 |
| 搜索不存在用户 | 是否展示错误状态 | 展示无结果提示 |
| 用户取消确认 | 是否改变展示好友 | 不保存选择，不改变原状态 |
| TOP1 免费机会 | 是否只允许终身 1 次免费 | 用完后再次设置 / 更换需按配置扣费 |
| TOP2 / TOP3 付费设置 | 是否按配置金币扣费 | 配置大于 0 时需确认并扣费 |
| 配置金币为 0 | 是否仍强制扣费 | 不扣费，可直接设置或更换 |
| 金币不足 | 是否允许更换成功 | 不允许更换，提示余额不足 |
| 扣费失败 | 原展示好友是否被清空 | 不改变原展示好友 |
| 被展示好友异常 | 是否继续正常展示 | 按策略隐藏、置灰或提示异常，不影响榜单主体 |
| 掉出 TOP3 后更换 | 是否仍可继续更换 | 不允许新增或更换，按保留策略展示原状态 |
